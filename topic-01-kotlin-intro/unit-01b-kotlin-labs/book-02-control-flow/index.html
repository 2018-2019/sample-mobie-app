if as a statement

Enter the following code into this new file:

fun main(args: Array<String>) {
    maxOf(4, 6)
}

fun maxOf(a: Int, b: Int) {
    val max: Int

    //traditional way to write if (as a statement)
    if (a > b) {
        max = a
        print("a is greater than b.  The max number is " + max)
    } else {
        max = b
        print("b is greater than a.  The max number is " + max)
    }
}

Run the code. You should have output similar to this:

b is greater than a. The max number is 6

This is the traditional form of if-statement that we are used to in Java.

Method with return type

We are going to rewrite the above code so that we don't store the maximum value in the local variable max...instead we will return the max value to the main method and print it out from there.

Refactor your code so that it now looks like this:

fun main(args: Array<String>) {
    println("The max number is: " + maxOf(4, 6));
}

fun maxOf(a: Int, b: Int) : Int {

    //traditional way to write if (as a statement)
    if (a > b) {
        return a
    } else {
        return b
    }
}

Note in the code above that we explicitly define a return type for a method by including a colon followed by a type i.e. : Int

When you run this code, you should have something like this printed to the console:

The max number is: 6

if as an expression

In Kotlin, the if is an expression, i.e. it returns a value. Therefore there is no ternary operator (condition ? then : else), because ordinary if works fine in this role.

As the if is an expression in Kotlin, we can rewrite our maxOf method like so:

//if as an expression
fun maxOf(a: Int, b: Int) =  if (a > b) a else b

Run this code and you should see there is no change in the output.

if as an expression (with code blocks)

When we first started this step, we printed out whether a was greater than b. We are going to put that code back in now into our if (as an expression).

Refactor your maxOf method so that your if expression has code blocks:

//if as an expression
fun maxOf(a: Int, b: Int) =
    if (a > b) {
        print("a is greater than b.  ")
        a
    }else {
        print("b is greater than a.  ")
        b
    }

When you run the code, you should now have the following output printed to the console:

b is greater than a. The max number is: 6

when

The when construct is equivalent to switch in Java, but it is more powerful. Type in the following code into your main class:

//when construct
fun printValue(value : Int){
    when(value){
        1 -> println ("The value is One")
        2 -> println ("The value is Two")
        in 3..5 -> println ("The value is between Three and Five")
        else -> {
            println ("The value is: " + value)
        }
    }

In particular, note the range operator e.g. 3..5 Also note that there is an else portion included. We should include our curly brackets with the else (a warning is generated otherwise).

Run this code by adding the following function call as the last statement in your main function:

    printValue(3)

The following output should be printed to your console:

The value is between Three and Five

when (no argument)

when can also be used as a replacement for an if-else chain.

If no argument is supplied, the branch conditions are simply boolean expressions, and a branch is executed when its condition is true.

Write the following new function and invoke it from main and invoke it using printAnimal("Guinea pig"):

fun printAnimal(aString: String){
    when {
        aString.equals("Dog") -> println("A dog")
        aString.equals("Cat") -> println("A cat")
        aString.contains("pig") -> println("A pig of some sort")
        aString.length == 0 -> println("Empty String..no Animal")
        else -> {
            println("No identifiable animal")
        }
    }
}

Outside 10..20

String Templates

Strings may contain template expressions, i.e. pieces of code that are evaluated and whose results are concatenated into the String.

A template expression starts with a dollar sign ($) and consists of either a:

  • simple name or
  • an arbitrary expression in curly braces

simple name

You should have this method in your main class (if you don't have it, cut and paste it in...and call it from the main function):

fun printValue(value : Int) = when(value) {
    1 -> "The value is One"
    2 -> "The value is Two"
    in 3..5 -> "The value is between Three and Five"
    6, 7, 8 -> "The value is either Six, Seven or Eight"
    else -> "The value is: " + value
}

We are going to convert the else code into using a String template. To do this, change the else code like so:

    else -> "The value is: $value"

Note how the $value is inside the double quotes. Compare the way this is written with the previous way.

Let's try it with another function. You should have this code in your codebase:

fun forEachStandard(){
    val items = listOf("apple", "banana", "kiwi")
    for (item in items){
        println("Item is: " + item)
    }
}

Change the println statement to use String templates (note that IntelliJ is probably giving you a warning and ALT+Enter might change it for you automatically):

    println("Item is: $item")

Now try use String templates in the println statement here:

fun forEachWithIndex(){
    val items = listOf("apple", "banana", "kiwi")
    for ((index, value) in items.withIndex()){
        println("The item at " + index + " is: " + value)

arbitrary expression

To demonstrate this construct, add this new function to your main class:

fun workingWithTemplates(){
    var firstString = "This is a String"
    var secondString = "and this is another String"

    //using simple names
    println ("Simple names: $firstString $secondString")

    //using arbitrary expressions in {}

Null Safety

In Kotlin, the type system distinguishes between references that can hold null (nullable references) and those that cannot (non-null references).

The Kotlin compiler makes sure you donâ€™t, by accident, operate on a variable that is null.

A non-null reference

Add a new function to your main class:

fun nullSafety() {
    var aString = "abc"
    val size = aString.length   //this is always going to have a reference
    aString = null              //syntax error
}

You cannot set a non-null reference (i.e. our String) to a null. This means that we cannot accidentally cause a null pointer exception.

Nullable references

But what if we need to set our String to a null? We will do this now by updating our variable declaration:

fun nullSafety() {
    var aString :String? = "abc"
    val size = aString.length   //now this is a syntax error
    aString = null              //compiles
}

Note that the aString = null statement compiles now, but we have caused a syntax error on line two of the function because variable aString can be null now.

If you need to have a nullable reference there are many ways around this.

Option 1 (basic if statement)

You can explicitly check if aString is null, and handle the two options separately. Try this now:

fun nullSafety() {
    var aString :String? = "abc"
    val size = if (aString != null) aString.length  else -1
    aString = null              
}

Option 2 (safe call operator ?.)

You can use the safe call operator ?.. This returns aString.length if aString is not null, and null otherwise. Try this now:

fun nullSafety() {
    var aString :String? = "abc"
    aString?.length 
    aString = null
}

Option 3 (not-null assertion operator)

You can use the !! Operator. This forces a call to our function and will return a non-null value of aString or throw an Null Pointer Exception if aString is null. Use sparingly! Try this now:

fun nullSafety() {
    var aString :String? = "abc"
    aString!!.length
    aString = null
}

Option 4 (Elvis operator)

If the expression to the left of the Elvis operator ?: is not null, the elvis operator returns it, otherwise it returns the expression to the right.

We will use this now in our nullSafety() function:

fun nullSafety() {
    var aString :String? = "abc"
    val size = aString?.length ?: -1
    aString = null
}
