Running the Hello World app

In this step, we will learn how to run an app on the virtual device (AVD). But we will have to set up the emulator first.

Setting up the emulator (AVD)

Now try to build and run the project (select Run -> Run app from the menu).

When you first attempt to run the app, you will be asked to create a Virtual Device. Click the Create New Virtual Device to do start this process.

  • Select a Phone...we used Nexus 5X. Click the Next button.

  • Download an API level...we chose Noguat(API Level 25).

  • When it is finished downloading, click the Finish button.

  • Then click on the downloaded release (the Next button should now be active). Click the Next button.

Your virtual device configuration should now look something similar to this one. Click the Finish button (and, if asked, accept any suggestions for installing additional items).

Installing HAXM

If it isn't already installed, we will need to install HAXM (Intel's Hardware Accelerated Execution Manager). It is a hardware-assisted virtualization engine that speeds up Android app emulation on a host machine.

To do this, select Tools-> SDK manager from the menu. Then check the box beside Intel x86 Emulator Accelerator (HAXM Installer). Click the Apply button.

Running the app

Gradle may continue and launch the emulator to run your app. If it doesn't select Run -> Run app from the menu again.

When your app launches successfully, it should look like this in the VM:

Viewing your AVDs

The VM that we just installed can be viewed by selecting Tools -> AVD Manager

The version, size and resolution are a reasonable compromise.

Logging Event Handling in Kotlin

We are going to be working exclusively in Kotlin - not Java. This affords considerable improvements, largely around conciseness and expressiveness of code.

Logging

To prepare for this, include this additional dependency in our build.gradle:

build.gradle (Module:app)

  implementation 'org.jetbrains.anko:anko-commons:0.10.7'

Your dependencies section will look like this now:

dependencies {
  implementation fileTree(dir: 'libs', include: ['*.jar'])

This library is documented here:

  • https://github.com/Kotlin/anko

We will be progressively introducing some of its features as we evolve the application.

The first feature is a simple way of logging:

Change the class to include the 'AnkoLogger' feature:

PlacemarkActivity

..
class PlacemarkActivity : AppCompatActivity(), AnkoLogger {
...

When you make this change, AnkoLogger will not be recognised - so you will need to import it. This can be triggered automatically (if you can figure this out...remember our Alt+Enter). The correct import statement is:

import org.jetbrains.anko.AnkoLogger

Then, in our onCreate() method, try it out:

...
    info("Placemark Activity started..")
...

Again, this will require an import - which will be this:

import org.jetbrains.anko.info

Now, make sure you can run the app and find the log entry in the 'Logcat' view in Android Studio:

Do not proceed further until you can locate something like the above in Logcat. The logs are a bit noisy, and may contain what look like errors. However, buried in there should be your message.

This style of logging is discussed here:

  • https://github.com/Kotlin/anko/wiki/Anko-Commons-%E2%80%93-Logging
